# Restricciones

## 3.2.1 Organizacionales

- El sistema de software deberá implementar su sistema de autenticación de usuarios utilizando protocolos seguros como **OAuth2** o **SAML 2.0** para la validación de identidad institucional.

## 3.2.2 Técnicas

- El sistema de software deberá proteger los datos personales de los usuarios mediante **cifrado de extremo a extremo (TLS 1.3)** para todas las comunicaciones y almacenamiento seguro de credenciales.
- El sistema de software deberá ser diseñado de tal forma que se permita la implementación de **i18n** para reglas, permitiendo la traducción de la interfaz a otros idiomas a partir del idioma español como base inicial.
- El sistema de software deberá operar correctamente en las **últimas dos versiones** (a la fecha 5 de septiembre del 2025) de los navegadores **Google Chrome, Firefox, Microsoft Edge y Safari**.

## 3.2.3 Legales o Normativas

- El sistema deberá cumplir con las **normas internas de la PUCP** en sus funciones y contenido, implementando medidas de moderación y reporte.
- El sistema deberá cumplir las **leyes nacionales sobre protección de derechos de autor y propiedad intelectual**, incorporando mecanismos de reporte y baja de contenido inapropiado.
- La interfaz de la plataforma deberá cumplir como mínimo con las pautas de accesibilidad **WCAG 2.1 Nivel AA**.
